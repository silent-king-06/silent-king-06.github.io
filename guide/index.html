🏷️ Tags（标签）使用指南

什么是标签？

标签是文章的关键词，一篇文章可以有多个标签。例如：

  • 一篇 React 教程可以有标签：React, 前端, 教程
  • 一篇读书笔记可以有标签：读书, Clean Code, 编程思想

📁 Categories（分类）使用指南

什么是分类？

分类是文章的层级归属，通常一篇文章只属于一个主分类。例如：

  • 技术分享 → 编程相关文章
  • 生活随笔 → 日常感悟
  • 读书笔记 → 书评和总结

🚀 常用命令速查

命令 作用
hexo new post "标题" 创建新文章
hexo new page "名称" 创建新页面
hexo server 或 hexo s 本地预览 (localhost:4000)
hexo generate 或 hexo g 生成静态文件
hexo deploy 或 hexo d 部署到远程
hexo clean 清除缓存和已生成文件
hexo clean && hexo g && hexo d 一键清理+生成+部署

✅ 写作 Checklist

发布文章前确认：

  • title 标题清晰明了
  • date 日期格式正确 (YYYY-MM-DD HH:mm:ss)
  • tags 添加了相关标签
  • categories 归类到合适分类
  • cover 设置了封面图（可选）
  • <!-- more --> 分隔了首页预览内容
  • 代码块指定了语言（如 ```python）
  • 图片路径正确
  • 本地预览无误 (hexo s)

💡 小贴士

  1. Markdown 文件名：使用英文和连字符，如 my-first-post.md
  2. 图片命名：避免中文和空格，如 project-screenshot.png
  3. 定期备份：source/ 目录包含你所有的内容，记得备份或推送到 GitHub
  4. 修改主题：CSS 文件在 source/css/cyberpunk-cosmos.css

祝你写作愉快！有问题随时查阅本指南。 ✨
